# Brief visual para trabajar en paralelo

## Frontera de trabajo

Codex deja congelada la capa funcional P0: permisos, Luma/Paisanos por evento,
acciones reales y Supabase. La identidad visual puede avanzar en paralelo siempre
que no cambie contratos de datos, rutas ni nombres de acciones sin coordinar.

## Lo que Claude puede explorar

- Sistema visual mas Paisanos y menos aerolinea.
- Login/magic link con imagen, marca y tono mas cuidado.
- Home tipo app diaria, no hero/presentacion.
- Paisaporte como objeto visual principal.
- Tratamiento para eventos Luma vs eventos Paisanos.
- Iconografia, color, tipografia, densidad y estados vacios.

## Evitar por ahora

- Cambiar nombres de tablas, campos o server actions.
- Rehacer navegacion/roles sin validar con P0.
- Introducir categorias de miembros, niveles o rankings.
- Hacer que Admin aparezca para miembros comunes.
- Convertir botones reales en mockups.

## Direccion

La app deberia sentirse como una herramienta privada de una comunidad creativa:
util, con caracter, clara para operar antes/durante/despues de un evento. El
Paisaporte puede usar la metafora de viaje, pero Luma cubre la operacion de
eventos; Paisanos debe sentirse como continuidad, identidad y red curada.
