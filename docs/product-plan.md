# Paisanos Members Club - Plan de producto

## Tesis

Convertir la comunidad Paisanos en una infraestructura propia de identidad,
eventos, conexiones y feedback. La V1 prueba si 20-30 miembros usan una
plataforma privada antes, durante y despues de un evento.

## V1

1. Auth por invitacion
2. Paisaporte digital
3. Eventos con modo por fuente: Luma o Paisanos
4. RSVP/check-in nativo solo cuando el evento no dependa de Luma
5. Admin basico para miembros, invitaciones y eventos
6. Feedback/propuestas en version minima
7. Analytics y fake doors para decidir V2

## P0 actual

1. Luma sigue siendo la fuente operativa segura para eventos existentes.
2. Paisanos App agrega identidad, contexto, directorio, radar y continuidad.
3. El admin no se muestra a miembros comunes; solo aparece para perfiles admin.
4. Los botones visibles deben navegar, escribir datos o explicar que son lectura.
5. La migracion de Luma a flujos nativos es evento por evento, no un corte abrupto.

## V2

1. Historial de participacion sin categorias de miembro
2. Directorio avanzado por intencion, aporte y disponibilidad
3. Matching mas rico por evento
4. Chat moderado por canales
5. Bolsa de trabajo interna
6. Sellos de hitos del Paisaporte, no rankings personales

## Later

1. Membresias pagas
2. Integracion HubSpot
3. Comunidad como panel de research
4. Paisaporte portable entre comunidades aliadas
5. Reputacion interna para contratacion

## Principios

- El evento fisico es la puerta de entrada al producto digital.
- El QR no es solo check-in: es identidad y continuidad.
- La participacion genera contexto e historial, no categorias ni jerarquias de miembros.
- La V1 mide interes antes de construir features pesadas.
- El admin tiene que ser operativo y sobrio, no una landing page interna.
